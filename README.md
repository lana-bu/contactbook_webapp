- Adding contacts to list
  - Prompt: "In my ContactList component, I want the web app to display a no contacts message when the items array is empty, but I want this message to be removed once a new contact is added, and I want that new contact to be displayed in its place. Currently, the no contacts message gets displayed, but when I try to add a contact, the server meets an error. How can I get the contactsDisplay variable to hold the updated contact list and display it properly?"
  - Response: "Two problems were fixed: 1) Typo — contactsDipslay was misspelled, so the actual contactsDisplay variable was never assigned in the else branch, staying undefined. 2) JSX concatenation — You can't use += to build up JSX. React expects an array of elements, which is exactly what .map() returns. By assigning the result of .map() directly to contactsDisplay, React will render the array of components properly."
- Search filtering without editing contactsData array
  - Prompt: "I am working on the search bar component for my web app. I want it to manipulate the contact list component by having it only display the contacts whose name or email contains the value being searched. I also want the list to update live (as the user is typing the search value) if possible. I am currently confused as to how it can filter the list without editing the actual contact data (it should not remove contacts that do not match from the contactsData array)."
  - Response: "Here's how it works: 1) contacts always holds the full, unmodified list of all contacts. 2) searchTerm holds whatever the user has typed in the search bar. 3) filteredContacts is computed fresh on every render using .filter(), which creates a new array containing only the matching contacts — it never mutates contacts. 4) ContactList receives filteredContacts instead of contacts, so it only displays matches. When the search field is empty, searchTerm is '', and every contact's name/email .includes('') is true, so the full list is shown. The filtering updates live as the user types because each keystroke calls onSearchChange, which updates searchTerm via setSearchTerm, triggering a re-render with the new filtered results."
- Errors in contact deletion code
  - Prompt 1: "I tried to implement contact deletion in my web app. I set it up so that when the delete button is pressed, the ID of the contact to be deleted gets sent from the contact items component to the contact list component and then to the app. In the app, a filter is performed on the contacts to update the contacts data with the chosen contact deleted. However, my web app is breaking when I try to add a contact now. Do you know what's wrong?"
  - Response 1: "The problem was that const declarations are not hoisted like function declarations. When a contact gets added, the else branch runs and tries to reference transferContactToDelete, but it hasn't been defined yet at that point in execution — causing a ReferenceError. Moving the definition above the if/else block resolves this."
  - Prompt 2: "That fixed the issue with the server not loading, but now the delete button won't work properly, and I can't tell why."
  - Response 2: "The problem was that key is a special React prop — it's used internally by React for reconciliation and is not accessible via props.key. So props.id was undefined, and the filter in deleteContactHandler couldn't match any contact. Adding an explicit id={item.id} prop makes it available to ContactItem."
